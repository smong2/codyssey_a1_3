# 📝 웹 서비스 기획서: [저기요.ai](http://저기요.ai)

## 1. 서비스 개요

- **서비스명**: 저기요.ai (Jogiyo.ai)
- **서비스 목적**: 매일 "오늘 뭐 먹지?"를 고민하는 사람들의 시간을 단축시켜주기 위해, 사용자가 원하는 조건(위치, 분위기, 가성비 등)을 입력하면 AI가 상황에 맞는 실존 맛집을 추천해 주는 웹 서비스입니다.
- **타겟 사용자**:
  - 점심 메뉴를 고민하는 직장인 및 학생
  - 낯선 동네에서 빠르게 맛집을 찾고 싶은 여행객/방문객
  - 데이트, 회식 등 특정 목적에 맞는 식당을 찾는 사용자

## 2. 페이지 및 섹션 구성

본 서비스는 SPA(Single Page Application) 형태로 동작하며, 자바스크립트를 이용해 3개의 주요 화면(섹션)을 전환 및 렌더링합니다.

1. **홈 (AI 검색 및 결과 섹션)**: 사용자 입력을 받는 검색바와 AI가 추천한 맛집 카드가 렌더링되는 메인 공간 (`#homePage`)
2. **즐겨찾기 섹션**: 마음에 드는 식당을 로컬 스토리지에 저장하고 모아보는 개인화 공간 (`#favPage`)
3. **이미지 모달 섹션**: 맛집 카드의 이미지를 클릭 시, 원본 크기로 확인하고 다중 이미지를 슬라이드 할 수 있는 갤러리 섹션 (`#imageModal`)

## 3. 핵심 기능 및 UX 고도화

- **다크모드 지원 (UX 고도화)**: 사용자의 눈 피로도를 줄이기 위한 테마 변경 기능 (CSS 변수 및 `localStorage` 활용)
- **마이크로 인터랙션 (UX 고도화)**: `alert()` 창 대신 자체 구현한 '토스트(Toast) 팝업'을 통해 부드러운 알림 제공
- **Discord 웹훅 (운영 자동화)**: 사용자가 검색을 수행할 때마다 관리자의 디스코드 채널로 검색어와 결과 수를 실시간 전송하여 유입 데이터 모니터링 (`ai.py` 내 구현)

## 4. AI 기능 설계 및 예외(실패) 처리 기준

Vercel Serverless Function(Python)과 Google Gemini API를 연동하여 AI 추천 엔진을 구현합니다.

- **입력 (Input)**: 텍스트 입력창을 통한 자연어 검색어 (예: "개포동역 가성비 한식", "강남역 조용한 이자카야")
- **출력 (Output)**:
  - AI가 검색어의 문맥을 파악하여 **실존하는 맛집 5곳**을 추천.
  - 각 맛집에 대해 식당명, 설명, 실제 주소, 카테고리, 네이버 검색 최적화 키워드를 JSON 객체로 반환.
  - 백엔드에서 네이버 이미지 검색 API를 2차로 호출하여 실제 매장/음식 사진 URL을 매칭하여 프론트로 전달.
- **예외 및 실패 처리 (Failover UX)**
  1. **빈 입력 (필수값 누락)**: 입력창이 비어있을 경우 서버로 요청을 보내지 않고 프론트엔드에서 "⚠️ 검색어를 입력해주세요!" 토스트 팝업 띄운 후 인풋 포커스.
  2. **지연/타임아웃**: LLM 처리 시간이 길어질 것을 대비해 `AbortController`로 15초 타임아웃 설정. 15초 초과 시 "요청 시간이 초과되었습니다. ⏳" 안내 표시.
  3. **API 오류 (4xx/5xx)**: Vercel 서버 에러 또는 AI/네이버 API 한도 초과 시 "서버 오류가 발생했습니다. 🚨" 안내 표시.
  4. **이미지 없음 (Fallback)**: 네이버 검색 결과 사진이 없을 경우 `via.placeholder.com`의 기본 대체 이미지를 제공하고 클릭(모달) 이벤트를 차단.

## 5. 기술 스택

- **Frontend:** HTML5, CSS3, JavaScript (Vanilla JS)
- **Backend:** Python (Vercel Serverless Functions)
- **Deployment:** Vercel
- **AI API:** OpenAI API (Gemini 3.1 pro)

## 6. 기대 효과

- '요기요'를 패러디한 친숙한 브랜드 네이밍으로 사용자에게 재미를 제공함.
- 단순 검색이 아닌 AI의 '이유 있는 추천'을 통해 사용자 의사결정의 신뢰도를 높임.
- 반응형 웹 구현을 통해 언제 어디서든 모바일로 간편하게 맛집 확인 가능.
