# [서비스 기획서] 저기요 (Jogiyo) - AI 맛집 큐레이터

## 1. 서비스 개요

- **서비스명:** 저기요 (Jogiyo)
- **서비스 목적:** "오늘 뭐 먹지?"라는 고민을 가진 사용자들에게 AI가 현재 위치와 상황에 맞는 최적의 식당과 메뉴를 추천하여 결정 시간을 줄여주고 즐거운 식사 경험을 제공함.
- **타겟 사용자:**
  - 결정 장애로 메뉴 선택에 어려움을 겪는 직장인 및 학생
  - 현재 위치 주변의 새로운 맛집을 찾고 싶은 탐험가
  - 배달보다는 직접 방문해서 먹는 '외식'을 선호하는 사용자

## 2. 페이지 구성 및 네비게이션

서비스는 GNB(Global Navigation Bar)를 통해 이동 가능한 2개의 페이지와 3개의 핵심 섹션으로 구성됩니다.

### 2.1. 페이지 구조

1.  **메인 페이지 (Home)**
    - **섹션 1: AI 검색 섹션** - 사용자로부터 위치, 상황, 음식 종류 등의 정보를 입력받는 영역.
    - **섹션 2: 추천 결과 섹션** - AI가 분석한 맛집 리스트가 카드 형태로 노출되는 영역.
    - **섹션 2-1: 인기 북마크 피드 (초기 화면)** - 검색 전, 메인 페이지 하단에 다른 사용자들이 많이 저장한 '인기 맛집' 리스트 노출
2.  **즐겨찾기 페이지 (Favorites)**
    - **섹션 3: 저장 목록 섹션** - 사용자가 '저장하기'를 누른 맛집들이 로컬 스토리지에 보관되어 표시되는 영역.

### 2.2. 네비게이션 (GNB)

- 상단 고정 GNB를 통해 '홈'과 '즐겨찾기' 메뉴 간의 자유로운 이동을 지원합니다.
- 반응형 설계를 통해 모바일 환경에서도 최적화된 메뉴 UI를 제공합니다.

## 3. 핵심 기능 (AI 기능 포함)

### 3.1. AI 맛집 추천 (핵심 기능)

- **기능 설명:** 사용자의 입력 데이터를 기반으로 OpenAI API(또는 설정된 AI)가 최적의 맛집 3곳을 추천합니다.
- **입력(Input):**
  1. **위치 (필수)**: 구체적인 지역명 (예: 강남역, 성수동)
  2. **음식 종류 (필수)**: 한식, 중식, 일식, 양식, 기타/분식, 카페/디저트
  3. **방문 상황 (필수)**: 혼밥, 데이트, 비즈니스, 모임, 가족외식
  4. **선택 항목 (펼치기)**: 가격대(세그먼트), 추가 요구사항(텍스트)
  5. **추가 요청 (선택)**: "주차 가능", "조용한 곳", "비건 메뉴" 등 자유 입력

- **출력(Output):**
  - 식당 이름, 대표 메뉴, AI의 추천 사유(큐레이션 멘트)
- **실패 처리 기준:**
  - **빈 입력:** 필수 값(위치) 누락 시 "어디로 가실지 알려주세요!" 안내 메시지 출력.
  - **API 오류:** 서버 통신 실패 시 "셰프 AI가 잠시 휴식 중입니다. 다시 시도해주세요." 메시지 노출.
  - **지연 처리:** 응답 대기 중 로딩 스피너와 함께 "저기요 AI가 맛집을 찾고 있어요..." 문구 표시.

### 3.2. 맛집 즐겨찾기

- AI가 추천해준 결과 중 마음에 드는 곳을 브라우저의 `localStorage`에 저장합니다.
- 페이지를 새로고침하거나 나중에 다시 접속해도 저장된 목록을 확인할 수 있습니다.

## 4. 기술 스택

- **Frontend:** HTML5, CSS3, JavaScript (Vanilla JS)
- **Backend:** Python (Vercel Serverless Functions)
- **Deployment:** Vercel
- **AI API:** OpenAI API (GPT-3.5/4)

## 5. 기대 효과

- '요기요'를 패러디한 친숙한 브랜드 네이밍으로 사용자에게 재미를 제공함.
- 단순 검색이 아닌 AI의 '이유 있는 추천'을 통해 사용자 의사결정의 신뢰도를 높임.
- 반응형 웹 구현을 통해 언제 어디서든 모바일로 간편하게 맛집 확인 가능.
